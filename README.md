# CURE Web

Устанавливаемая веб-версия CURE с общей клиникой, аккаунтами сотрудников и синхронизацией пациентов, визитов, фотографий и финансов.

## Что работает

- PWA: установка из Safari на экран iPhone.
- Адаптивный интерфейс для iPhone, iPad и компьютера.
- Email + пароль через Supabase Auth.
- Единая клиника с кодом приглашения.
- Отдельные аккаунты сотрудников и общая база.
- Realtime-обновления между открытыми устройствами.
- Пациенты, подробный анамнез и стоматологическая карта.
- Визиты, стоимость, оплата, скидка, возврат и долг.
- Фото из галереи или камеры, закрытое хранилище.
- Доходы, расходы, графики и CSV.
- Светлая и тёмная тема.
- Демо-режим через localStorage до подключения Supabase.

## Почему не один пароль на всех

Приложение допускает вход одним аккаунтом на нескольких устройствах, но для клиники безопаснее:

1. Владелец создаёт клинику.
2. Передаёт коллеге код приглашения.
3. Коллега регистрирует собственный аккаунт.
4. Оба аккаунта видят одну и ту же базу.

Так можно отозвать доступ конкретному сотруднику, не меняя пароль всей клинике.

## Локальный запуск

```bash
pnpm install
pnpm dev
```

Без `.env` откроется полноценный локальный демо-режим.

## Подключение синхронизации

1. Создайте проект на https://supabase.com.
2. Откройте `SQL Editor`.
3. Выполните целиком файл `supabase/schema.sql`.
4. В `Project Settings → API` скопируйте Project URL и publishable/anon key.
5. Скопируйте `.env.example` в `.env`:

```env
VITE_SUPABASE_URL=https://YOUR_PROJECT.supabase.co
VITE_SUPABASE_ANON_KEY=YOUR_PUBLISHABLE_ANON_KEY
```

6. Перезапустите `pnpm dev`.

Никогда не добавляйте `service_role` key в веб-приложение.

## Развёртывание

Подойдут Vercel, Netlify или Cloudflare Pages.

Команда сборки:

```bash
pnpm build
```

Папка публикации:

```text
dist
```

После публикации добавьте домен в Supabase:

- `Authentication → URL Configuration → Site URL`
- `Authentication → URL Configuration → Redirect URLs`

Для production настройте собственный SMTP в Supabase. Встроенная тестовая отправка писем имеет низкие лимиты.

## Установка на iPhone

1. Откройте опубликованный HTTPS-адрес в Safari.
2. Нажмите «Поделиться».
3. Выберите «На экран Домой».
4. Нажмите «Добавить».
5. Запускайте CURE с иконки как обычное приложение.

## Медицинская приватность

RLS ограничивает строки участниками клиники. Фотографии хранятся в private bucket и открываются по временным signed URL. Перед реальным использованием с пациентами требуется определить применимые требования законодательства, договор с облачным провайдером, правила согласия пациентов, резервного копирования и сроков хранения.
